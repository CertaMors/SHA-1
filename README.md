# SHA-1
Implementation of the sha-1 algorithm

Представлена програмна реалізація алгоритму SHA-1 демонструє фундаментальний
принцип забезпечення цілісності інформації в сучасних системах управління. В контексті
інноваційної економіки, де дані є ключовим активом, використання хеш-функцій виступає
гарантом незмінності транзакцій та достовірності електронного документообігу. Хоча на
сьогодні SHA-1 вважається криптографічно вразливим для колізійних атак і в новітніх
платіжних системах заміщується стандартами сімейства SHA-2, розуміння його
архітектури є критичним для підтримки легасі-систем та розуміння еволюції
інфраструктури відкритих ключів (PKI). Реалізований механізм згортання повідомлення
довільної довжини у 160-бітний дайджест ілюструє технологічну базу для створення
цифрових підписів та сертифікатів, що є основою довіри в цифровій економіці.»

З методологічної точки зору, даний код є практичною верифікацією детермінованого
алгоритму, описаного в стандарті FIPS 180-1. Реалізація базується на суворій математичній
послідовності: попередній обробці вхідних даних (padding), розбитті на 512-бітні блоки та
виконанні 80 ітерацій циклічних зсувів та логічних операцій (XOR, AND, NOT). Цей код
дозволяє дослідити "лавинний ефект" хеш-функції, де зміна навіть одного біта вхідного
повідомлення призводить до
кардинальної зміни вихідного дайджесту, що є ключовою вимогою до криптостійкості. У
контексті патентоведення, дана робота підкреслює відмінність між абстрактною
математичною ідеєю, яка зазвичай не підлягає патентуванню, та її конкретною
програмною реалізацією, що може бути об’єктом авторського права як комп'ютерна програма.
